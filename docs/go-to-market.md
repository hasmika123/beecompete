# BeeCompete — Go-to-Market

**Status:** Living document · **Last updated:** 2026-07-07 · Depends on: `development-process.md` (release ladder), `monetization.md`, `competitive-analysis.md`, `phase-1-plan.md` (seeding)

Code and catalog make the product; this doc is the plan for **demand** — the non-coding work that
runs in parallel with every build phase. If it isn't scheduled here, distribution won't happen by
itself.

---

## 1. Positioning (one line per audience)

- **Students / parents:** "Every competition worth entering — one place, always current."
- **Educators:** free tools to discover, assign, and track competitions for a whole class or club.
- **Hosts / fairs:** a modern toolkit *plus* a built-in audience — transparent pricing, 0% entry-fee cut.

## 2. Channel plan by release

| Release | Primary channels | Notes |
|---|---|---|
| **R0 — Waitlist** | Landing page + Brevo email capture; shares in parent/educator communities | Starts before code. |
| **R1 — Browse-only marketplace** | **SEO is the engine**: per-competition + per-category landing pages; seeding order = keyword-research order (feeds task S2 in `phase-1-plan.md`); sitemaps to Google/Bing; launch shares in niche communities (homeschool groups, gifted-ed forums, teacher subreddits/FB groups); first digest to the waitlist | Traffic + indexing are R1's success measures. |

| **R4 — Host Tools (science-fair wedge)** | **Direct outreach** — see §3; warm list built since Phase 1 | Relationship-driven B2B, not SEO. |

## 3. Science-fair wedge outreach 🛑 *(starts NOW — Phases 1–2, non-coding)*

The Phase-3 wedge (`monetization.md` §8) depends on relationships built **early**. The
Scienteer-displaced states (**TX, AL, LA, ME, MO, VT, VA**) are in transition and the window is
closing (see `competitive-analysis.md` §5).

- **Who:** regional/state fair directors, school science-fair coordinators, and SRC chairs in the
  displaced states. Contacts via state fair sites and the Society for Science affiliate directory.
- **Goals:** (1) validate demand + willingness to pay, (2) learn the real feature bar first-hand
  (what STEM Wizard misses; what Scienteer got right), (3) recruit **2–3 design partners** for
  Phase 3, (4) build a warm launch list for R4.
- **How:** short calls/emails, not surveys. A few conversations per month is enough.
- **Output:** interview notes are a **prerequisite input to Gate A — the science-fair wedge
  deep-dive** (`development-process.md` §6a). Phase-3 design does not start without them.

## 4. Success metrics & gates *(fill in targets as each release approaches)*

- **R1:** organic traffic trend; impressions/clicks on the top-50 listings (Search Console); waitlist size.
- **R2:** visit→signup conversion; save/track rate; weekly retained tracker users.
- **R3:** free→Participant+ conversion; revenue per competition.
- **R4:** design partners signed; fairs onboarded from the displaced states.

## 5. Open GTM items

- **Problem interviews** (`vision-prd.md` §7.1): 8–12 each with students, parents, educators — run during R0/R1; **required input to the R2 go/no-go validation gate** (`phase-1-plan.md` sequencing notes).
- Keyword-research pass (feeds seeding order, task S2).
- Community list for launch shares (collect as we go).
- Referral program design (M27, Phase 2).
- Content marketing / editorial guides (M19, Phase 2).
